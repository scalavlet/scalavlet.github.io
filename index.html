Scalavlet

Scalavlet is yet another tiny Sinatra like Scala web framework on top of Servlet specification. That provides a simple coding way for HTTP request routing without any XML configuration and complicated Scala implicit conversions. 

object SampleServlet extends Scalavlet {
  get("/ping") { request =>
    "pong"
  }
}

class Bootstrap extends Bootable {
  def onStart (ctx: Context): Unit = {
    ctx.mount(SampleServlet, "/")
  }
}

Intuitive Routing

Sinatra like routing is very easy to read, nothing special Scala implicit techniques are used. Routing part is highly affeceted by Scalatra. more ...

No XML Configuration

Instead of dependeing on Servlet's web.xml for routing and parameter getting/setting, Scalavlet chose programatic building and the centralized Typesafehub/Config based configuration. more ...

Easy to Start

Scalavlet is not a full-stack web framework, but provides several sample implementations for typical usages. You can get ready soon. more ...

Be RESTful

One of the greatest features of Scala is DSL. Scalavlet integrates with json4s which provides beatiful JSON building DSL. For request unmarshalling, Jackson is used. more ...

Async with Future

Since 3.0, Servlet supports asynchronous processing feature. When Scalavlet detects a Future object as aresponse, it automatically starts the Servlet async mode accordingly. more ...
